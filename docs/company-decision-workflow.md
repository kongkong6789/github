# 公司级辅助决策工作流

本项目已经从 SKU/库存级决策扩展到公司经营层面的辅助决策。

## 已新增 Agent

- `quality_gate_agent`：数据质量门。先判断数据是否足够可靠，避免脏数据直接进入决策。
- `company_strategy_agent`：公司经营策略。综合公司背景、产品线、库存、销售、广告、财务、供应商和历史决策。
- `financial_planning_agent`：财务规划。关注收入、成本、毛利、现金流、广告支出、库存占用和采购承压。

## 已新增工具

- `assess_data_quality`：检查字段完整性、空值、重复数据和关键数据缺口。
- `analyze_company_financial_position`：生成公司级财务和现金流快照。
- `analyze_company_strategy`：输出公司级经营问题、库存风险、数据缺口和动作优先级。

## 推荐资料目录

```text
<A2A_PROJECT_ROOT>/raw
├─ 公司背景
├─ 财务
├─ 库存
├─ 销售
├─ 广告
├─ 供应商
├─ 平台规则
└─ 历史决策
```

这些资料会进入：

```text
<A2A_PROJECT_ROOT>/wiki
<A2A_PROJECT_ROOT>/data/cleaned
```

## 前端推荐说法

```text
请处理 raw 目录里的公司资料，按照 Karpathy LLM Wiki 的方式整理成 Obsidian 知识库，并基于公司背景、库存、销售、广告、财务和供应商资料，生成公司经营层面的辅助决策报告。请先清洗表格，再做数据质量检查，不要跳过缺口提示。
```

也可以更直接：

```text
基于当前 cleaned 数据和 Obsidian 知识库，帮我做公司级经营辅助决策：下个月应该优先做哪些动作，哪些产品线要投入，哪些库存要控制，现金流有什么风险。
```

## 输出应包含

- 公司级结论。
- 数据来源和证据链。
- 数据质量等级和缺失字段。
- 库存、财务、广告、供应商、产品线风险。
- 推荐动作优先级。
- 保守、平衡、激进方案。
- 需要老板或人工确认的事项。

## 边界

系统只能做辅助决策，不能替代老板、财务负责人、法务或税务顾问。以下动作必须人工确认：

- 大额采购。
- 融资、贷款、担保。
- 税务和合同。
- 裁员、薪酬、股权。
- 外发消息或创建真实订单。
