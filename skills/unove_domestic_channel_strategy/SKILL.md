# UNOVE 国内渠道经营策略

## 适用范围

本页面作为 UNOVE 国内渠道经营分析、月度计划拆解、分销/线下跟进、竞品监控和 Agent 决策输出的长期业务规则。后续 Agent 做渠道建议时，必须先引用本页面的边界，再结合 DuckDB、wiki/LightRAG、必要时只读 ERP 数据给出结论。

## 平台定位和不做事项

### 天猫

- 不做店播。
- 达播只考虑李佳琦，其他达播不做。
- 李佳琦一年预计上 1-2 次，不能把它当成日常增长抓手。
- 天猫每天都有活动，核心靠日销和推广。
- 分析天猫时重点看：日销稳定性、推广效率、活动节奏、价格体系、库存覆盖和主推款承接能力。

### 天猫国际

- 受平台流量分配机制影响，大贸天猫整体会比天猫国际更容易跑出来。
- 天猫国际若要跑出来，必须和大贸天猫错开款式，不能直接做同款内耗。
- 分析天猫国际时重点看：款式差异化、价格带、库存深度、是否和大贸渠道互相挤压。

### 抖音

- 每个月需要按 200 万+ GMV 目标拆解。
- 主要靠达播和千川投放。
- 分析抖音时重点看：达播排期、头部达人产出、千川消耗、净 GMV、ROI、费比、货盘承接、库存和复购。

### 快手和小红书

- 快手达播不稳定，主要看辛选达播上播频次。
- 小红书受类目和平台特点影响，整体不作为主要成交平台。
- 小红书更适合作为舆情、种草声量和竞品动作观察来源，不应默认给成交增长建议。

### 拼多多

- 拼多多主要做百补。
- 分析拼多多时重点看：价格力、百补坑位、毛利安全线、库存周转和平台补贴节奏。

### 京东自营

- 京东自营主要做价格。
- 分析京东时重点看：价格竞争、活动价、供货节奏、毛利和库存占用。

### 不纳入建议的渠道

- 得物和唯品会暂不考虑。
- 天猫超市不需要给经营建议。

## 组织跟进规则

### 分销组和外贸组

- 核心任务是跟进销售进度。
- 可以提供竞品 C 店拓展建议，但不要推荐已经卖 UNOVE 的店铺作为拓展对象。
- 应输出销售进度、客户/店铺拓展清单、竞品 C 店观察、拿货政策建议和下次跟进动作。

### 线下组

- 核心任务是跟进销售进度和客户分层。
- 根据客户名单识别高价值客户、沉默客户、很久没下单客户和需要催促下单客户。
- 客户名单需要持续更新，分销客户也要纳入。
- 应形成分销和线下客户 list、销售排名、客户分层、催单名单和拿货政策建议。

## 营销和竞品监控

- 每月跟踪小红书关于 UNOVE 的笔记数量、内容主题、舆情变化和负面反馈。
- 重点关注竞品动作：offrelax、且初、巴黎欧莱雅。
- 小红书分析默认服务于声量、舆情和竞品观察，不默认服务于成交转化。

## 计划拆解

- 每月先按渠道目标分配，再拆解到任务、资源、数据缺口和责任动作。
- 抖音必须明确 200 万+ GMV 目标的达播/千川拆解。
- 天猫必须明确日销和推广承接。
- 分销、外贸、线下必须明确客户 list、销售排名、沉默客户和催单动作。

## Agent 输出要求

Agent 在引用本策略时，输出必须包含：

1. 渠道定位和不做事项。
2. 本月目标和核心抓手。
3. 需要读取的数据来源：DuckDB mart、wiki/LightRAG、客户名单、只读 ERP。
4. 分渠道执行清单。
5. 分销/外贸/线下客户跟进清单。
6. 营销舆情和竞品监控清单。
7. 数据缺口和需要人工确认的问题。

## 数据优先级

- 默认先读本地 DuckDB mart 和 wiki/LightRAG。
- 当用户明确要求实时 ERP，或本地 mart 缺少销售/库存/客户关键证据时，才调用吉客云/金蝶只读 ERP。
- 实时 ERP 查询结果必须标注为 live_read_only_fallback，不能当成已入库事实。
