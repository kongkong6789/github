# 吉客云 ERP Skill 用户端 SOP

适用对象：运营、客服、仓储、财务等需要通过对话操作吉客云 ERP 的同事。  
使用原则：把业务需求说清楚，由 Skill 查询、校验、预览、创建或修改单据。

## 1. 基本使用规则

1. 所有新建、修改、审核、驳回类操作，必须先看 Skill 返回的确认信息。
2. 确认信息无误后，再回复“确认创建”“确认修改”“确认审核”等明确指令。
3. 不确定需要提供什么字段时，直接说：“给我模板”。
4. 不确定渠道、仓库、货品、业务员名称时，可以让 Skill 先查询候选。
5. 单据创建成功后，必须核对 Skill 反查回来的单据信息，不只看单号。
6. 如果 Skill 返回 `needs_input`，说明信息不完整或有风险，本次不会创建单据；按模板补齐后再继续。
7. 如果发现 Skill 识别错了，请直接说明“错在哪里、正确值是什么”。Skill 会记录纠错经验，避免后续同类问题重复出现。

## 3. 销售单创建 SOP

建议先体检再创建，尤其是批量建单、陌生仓库、陌生渠道或不确定批次时。

```text
先帮我预检这个销售单，不要创建：渠道【】、仓库【】、业务员【】、客户名称【】、货品【】数量【】。
```

Skill 会先返回最终识别到的渠道、仓库、物流、业务员/登记人、批次库存和阻断问题。只要存在阻断问题，Skill 会返回模板并停止创建；确认无误后再创建。

销售单分三类：

| 类型 | 适用场景 | 金额规则 | 标记 |
|---|---|---|---|
| 寄样单 | 给客户寄样品 | 通常为 0 | 样品 |
| 补发单 | 漏发、破损、补寄 | 通常为 0 | 补发 |
| 普通手工销售单 | 手工销售、收费订单 | 必须提供单价和金额 | 无 |

### 3.1 单个销售单必填信息

| 字段 | 是否必填 | 说明 |
|---|---:|---|
| 单据类型 | 必填 | 寄样、补发、普通手工销售单 |
| 销售渠道 | 必填 | 渠道必须和仓库属于同一公司 |
| 仓库 | 建议提供 | 不提供时系统按渠道默认仓库匹配 |
| 客户名称 | 寄样必填 | 寄样单必须由用户提供客户名称 |
| 收件人 | 必填 | 收货人姓名 |
| 手机号 | 必填 | 收货手机号 |
| 收货地址 | 必填 | 建议包含省、市、区、详细地址 |
| 货品 | 必填 | 货品编号、条码或货品名称 |
| 数量 | 必填 | 正整数 |
| 单价/金额 | 普通手工单必填 | 寄样、补发通常为 0 |
| 业务员 | 必填 | 必须填写本人员工姓名；系统同步写入登记人 |
| 批次 | 有批次管理时必填或确认推荐 | 可让系统先查库存并推荐 |
| 备注 | 可选 | 区分客户备注和内部备注 |

### 3.2 推荐输入模板

```text
帮我新建一个【寄样单/补发单/普通手工销售单】
销售渠道：
仓库：
客户名称：
收件人：
手机号：
地址：
货品：
数量：
单价：
业务员：
批次：
客户备注：
内部备注：
是否提交审核：
```

### 3.3 确认与核对

创建前，必须核对：

- 单据类型是否正确。
- 渠道、仓库、物流是否正确。
- 客户名称、收件人、手机号、地址是否正确。
- 货品、规格、数量、批次是否正确。
- 业务员和登记人是否为本人姓名。
- 普通手工单的单价、金额是否正确。

创建后，必须核对 Skill 返回的反查字段：

- 销售单号。
- 渠道、仓库、物流公司。
- 客户名称、收件人、手机号、地址。
- 业务员、登记人。
- 标记：样品或补发。
- 货品明细、数量、单价、金额、批次。
- 当前状态和后续处理提示。

## 4. 仓库与物流默认规则

如用户提供仓库，系统按最终仓库匹配物流：

| 仓库名称包含 | 默认物流 |
|---|---|
| 麦歌 | 麦歌中通 |
| 宝鼎 | 宝鼎中通 |
| 韩国申通 | 依然物流 |
| 韩国韵达 | 韩国韵达-韵达国际 |

如匹配结果不符合实际业务，必须在创建前指出并要求修改。

## 5. Excel 批量建单 SOP

### 5.1 操作步骤

1. 上传 Excel 或 CSV。
2. 先说：“先预览，不要创建”。
3. 检查 Skill 返回的列映射、跳过行、错误原因和订单数量。
4. 修正表格或补充默认值。
5. 确认无误后，再说：“确认创建”。
6. 创建后核对每个成功单据的反查信息。

### 5.2 官方模板支持字段

常见可识别表头：

| 表头 | 含义 |
|---|---|
| 网店订单号 | 外部订单号；多行同号会合并为一个订单 |
| 收货人 | 收件人 |
| 手机 | 收件手机号 |
| 省份、市（区）、区（县）、收货地址 | 收货地址 |
| 发货仓库 | 仓库 |
| 销售渠道名称 | 销售渠道 |
| 货品名称、条码、货品编号、规格 | 货品信息 |
| 批次号 | 指定批次 |
| 数量、单价、金额 | 明细数量与金额 |
| 物流公司 | 指定物流 |
| 业务员 | 员工姓名 |
| 标记 | 样品、补发、普通 |
| 客户账号/客户名称 | 客户名称 |
| 客服备注 | 内部备注 |
| 客户备注 | 客户备注 |

注意：

- 表头里的中文括号、英文括号、空格、全角半角差异，系统会尽量自动识别。
- 表格没有业务员时，必须在对话里提供默认业务员姓名。
- 寄样单没有客户名称时，必须在对话里提供默认客户名称。
- 批量导入建议先按 1 到 3 行测试预览，再批量创建。
- 预览模式会做建单前体检：渠道、仓库、物流、业务员/登记人、客户名称、金额、批次库存有问题会先阻断，不会直接创建。

## 5.3 纠错与复用

如果创建或查询结果不对，请用下面格式反馈：

```text
这个结果不对：错误是【】；正确应该是【】；以后同类场景请按【】处理。
```

Skill 会记录原因和防错规则。纠错后需要重新预检，预检通过前不会创建单据。

## 6. 批次库存 SOP

适用场景：

- 用户不知道该选哪个批次。
- 系统提示某批次无库存。
- 同一货品需要按多个批次发出。

推荐说法：

```text
查一下【仓库】里【货品编号/条码/货品名称】的批次库存和效期，数量需要【数量】，帮我推荐批次。
```

默认规则：

- 未指定批次时，系统默认按 FIFO 先进先出选择批次。
- 如果用户提出效期、生产日期、指定/排除批次等要求，系统会先筛选符合要求的批次，再按 FIFO 自动分配。
- 一个批次不够时，系统会自动拆分多个批次；总可用库存不足时才会提示缺货。
- 用户想自定义时，可以要求 Skill 展示候选批次后手动选择。

核对重点：

- 仓库是否为最终发货仓。
- 推荐批次是否有足够可用库存。
- 效期是否符合业务要求。
- 如系统推荐多个批次，确认拆分数量是否正确。

如果客户要求库存不足也先建销售单：

- 必须明确说：“允许先建待配批次单，等有库存后再匹配批次发货”。
- Skill 会创建未审核单，不会自动审核发货。
- 后续库存到货后，再让 Skill 查询批次并匹配批次，然后审核/递交仓库。

## 7. 调拨单 SOP

### 7.1 必填信息

| 字段 | 说明 |
|---|---|
| 调拨类型 | 同主体调拨或跨公司调拨 |
| 调出仓库 | 发出仓 |
| 调入仓库 | 接收仓 |
| 货品 | 货品编号、条码或货品名称 |
| 数量 | 正整数 |
| 批次 | 有批次管理时需确认 |
| 调拨原因 | 按系统字典值填写 |
| 备注 | 用户提供的调拨备注 |
| 申请人 | 默认使用当前操作人 |

### 7.2 推荐输入模板

```text
帮我新建调拨单
调拨类型：
调出仓库：
调入仓库：
货品：
数量：
批次：
调拨原因：
备注：
申请人：
```

创建后必须核对：

- 调拨单号。
- 调出仓、调入仓。
- 申请人和部门。
- 调拨原因、备注。
- 货品、数量、批次。
- 单位是否为 `Pcs`。

注意：

- 新建调拨单必须让 Skill 走调拨工作流，不要让它手动拼 `erp.allocate.create`。
- 未指定批次时，系统默认按调出仓 FIFO 先进先出自动生成批次。
- 如果用户明确允许缺货也先建调拨单，需要说：“允许缺货先建调拨单，后续有库存再匹配批次”。此时系统只标记批次管理，不会伪造不存在的批次。

## 8. 查询与审核 SOP

### 8.1 查询销售单

推荐说法：

```text
查一下销售单 YR260429001034，返回完整字段。
```

常见核对字段：

- 单号、状态、渠道、仓库、物流。
- 收件人、手机号、地址。
- 货品、数量、批次。
- 业务员、客户名称、备注、标记。

### 8.2 查询基础资料

基础资料包括仓库、渠道、货品、用户等。

推荐说法：

```text
查询所有名称包含“分销组”的仓库，有“除分销组”的不要。
```

```text
查一下名称包含“Arencia”的渠道，按公司分组列出来。
```

注意：

- “包含/排除关键词”类需求会优先查本地全量缓存。
- 如果缓存不完整，Skill 会自动分页刷新后再过滤；按单个仓库名称或仓库编号建单时，如果第一页缓存没找到，也会自动全量刷新后再判断。
- 导出仓库批次库存报表时，仓库关键词、排除关键词、销量为空填0还是留空，都要按本次需求确认；分销组只是一个示例，不是固定条件。
- 不需要让同事自己写 Python、Node 或签名脚本；Skill 内置了可直接调用的签名工具和 API client。

### 8.3 待审核网店订单诊断

适用场景：平台订单下载到 ERP 后，没有被自动审核，需要运营判断原因。

推荐说法：

```text
帮我诊断【渠道/时间范围】的待审核网店订单，看看为什么没有自动审核。
```

Skill 会按以下原因分组：

- 缺货或批次库存不足。
- 仓库选错或缺少仓库。
- 物流缺失或不匹配。
- 退款、售后、退货风险。
- 无效货品、货品编号缺失或数量异常。
- 收件人、手机号、地址不完整。

处理原则：

- 可审核的订单先展示候选，用户确认后再批量审核。
- 仓库/物流错误先修改，再重新诊断。
- 缺货订单可以等待库存，或用户明确允许时先保留待配批次状态。
- 退款/售后/无效货品订单不要直接审核，先确认平台状态。

### 8.4 货品销售多维分析报表

适用场景：查询某个月或某个时间段，某些渠道的所有货品销售额、销量、发货量、退货量、退款金额、成本、毛利等指标。

推荐说法：

```text
导出 2026年5月 渠道A、渠道B 的所有货品销售额和销量，字段要全。
```

注意：

- 默认按发货时间统计；如需按下单时间或付款时间，需要明确说明。
- 默认按“渠道+货品”汇总；如需按规格、时间、仓库、业务员等维度，需要明确说明。
- 用户说渠道名称即可，Skill 会自动解析为吉客云 `shopIds`。
- 导出字段默认包含官方接口返回的全部字段，不只包含销售额和销量。
- 如果导出为空，但吉客云前台确认有销量，先核对当前 Skill 使用的 AppKey 是否是已订阅并有该公司、部门、渠道数据权限的应用。
- 如果多维报表接口持续返回空，需要由 IT 提供已开放平台查询的吉智BI自定义报表 ID。没有报表 ID 时，不要把空表当作可用结果上线。

### 8.5 渠道销售数量/金额汇总

适用场景：运营经常要看某个渠道或多个渠道，在某个月/时间段内的销售数量和销售金额。

推荐说法：

```text
汇总 2026年5月 渠道A、渠道B 的销售数量和销售额，按渠道+货品维度。
```

```text
汇总昨天发货的渠道名称包含分销组的销售数据，订单状态为发货在途或已完成，按渠道维度。
```

注意：

- 只要数量和金额时，默认走 `run_channel_sales_summary_workflow()`，不逐页拉销售单再本地汇总。
- 维度可以说“按渠道”、“按渠道+货品”、“按渠道日销”或“按渠道+货品日销”。
- 渠道可以直接说关键词，例如“渠道包含分销组”；Skill 会先匹配渠道并传 `shopIds` 给报表接口。
- 默认按发货时间统计；如需下单时间或付款时间，请明确说明。
- 如果报表返回 0 行但前台确认有数据，先让 IT 核对 AppKey 数据权限；也可以提供吉智BI自定义报表 ID 给 Skill 兜底查询。

### 8.6 入库/出库申请单

适用场景：创建吉客云入库申请单或出库申请单。

推荐说法：

```text
帮我新建出库申请单，申请人张三，仓库A，其他出库，货品G1数量10，批次按先进先出。
```

注意：

- 申请人必须提供员工姓名，Skill 会写入 `applyUserName`。
- 出库申请单不提供批次时，默认按仓库库存 FIFO 自动分配；有批次条件时，先按条件筛选再 FIFO。
- 入库申请单是新入库批次，只有用户提供批次号、生产日期、到期日期等信息时才写入。
- 信息不完整时，Skill 会返回模板，不会直接创建脏数据。

### 8.7 生成模板文件

适用场景：按销售单、库存、仓库等吉客云数据，生成固定 Excel 模板文件。

推荐说法：

```text
帮我按销售单 YR260429001022 生成申通仓出库单。
```

注意：

- 如果是新模板，先上传模板文件，并说明哪些单元格/列要填、哪些必须留空。
- 合并单元格只需要告诉 Skill 目标区域或目标显示位置，Skill 会写入合并区域左上角。
- 如果本次形成了稳定新流程，Skill 会默认建议单独打包成新 skill，避免当前吉客云 ERP skill 后续升级或重装时覆盖你的自定义流程；只有确认是通用基础能力时，才建议并入当前 skill。

### 8.8 审核或驳回

审核、驳回、批量修改仓库物流前，必须让 Skill 先列出候选并确认范围。

推荐说法：

```text
查一下今天待审核销售单，按渠道和仓库汇总，先不要审核。
```

```text
驳回销售单 YR260429001034，原因：客户信息错误。先让我确认。
```

## 9. 常见问题

| 问题 | 处理方式 |
|---|---|
| 不知道要给哪些字段 | 说“给我模板” |
| 渠道、仓库、货品名称不确定 | 让 Skill 先查候选 |
| 英文括号和中文括号不一致 | 可直接输入，系统会自动归一化匹配 |
| 批次提示没库存 | 让 Skill 查最终仓库的批次库存并推荐 |
| 寄样单创建失败 | 先检查是否提供了客户名称 |
| Excel 预览跳过行 | 根据错误原因补齐手机号、业务员、客户名称、数量等字段 |
| 创建成功但网页端看起来不对 | 以 Skill 反查字段为准，再核对吉客云页面；必要时提供单号重新查询 |

## 10. 禁止事项

- 不要在未预览的情况下要求直接批量创建。
- 不要用收件人代替寄样单客户名称。
- 不要让系统随便猜批次。
- 不要让他人姓名作为自己的业务员或登记人。
- 不要在未确认候选范围的情况下批量审核、驳回或改仓改物流。
- 不要只看创建接口返回的单号，必须看反查后的单据字段。
